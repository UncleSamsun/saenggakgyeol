Mac capture app

메모와 녹음 transcript를 시간순으로 맞춰 생각의 맥락을 복원한다.

생각결은 일반 노트앱이 아니라 세션 기반 Mac 앱입니다. 회의나 혼자 생각하는 중에 흐름대로 쓴 메모를 녹음 transcript와 대조해, 나중에 “왜 이 생각을 했는지”까지 되살립니다.

한눈 요약

생각결의 첫 버전은 “빠른 노트앱”이 아니라 “시간이 붙은 생각 세션을 정리하는 앱”이어야 합니다.

제품 한 줄

회의나 혼자 생각하는 세션에서 쓴 메모와 녹음 transcript를 시간순으로 맞춰, 나중에 맥락까지 복원해주는 Mac capture app.

핵심 차별점

“말한 내용을 정리”가 아니라 “내가 직접 적은 생각과 그 시점의 발화 맥락을 붙여 다시 이해하게 해주는 것”.

기본 사용

단축키로 sticky capture note를 열고 자유롭게 타이핑합니다. 녹음은 필요할 때만 켭니다.

저장 원칙

사용자가 소유하고 읽는 원본은 Obsidian Markdown입니다. 앱 DB는 인덱스와 캐시입니다.

문제 정의

회의나 생각 중에는 메모를 흐름대로 남기지만, 나중에 보면 왜 그 문장을 썼는지 맥락이 사라집니다. 녹음을 따로 남겨도 transcript와 내 메모가 분리되어 있으면 다시 대조하는 비용이 큽니다.

생각결은 세션 중 작성한 메모 chunk에 자동으로 시간을 붙이고, 녹음이 켜져 있으면 transcript chunk와 맞춥니다. 세션 종료 후 AI가 이 둘을 함께 읽어 summary, decisions, tasks, questions, evidence snippets를 만듭니다.

1. Open전역 단축키 또는 메뉴바에서 sticky capture note를 엽니다.
2. Write세션 내내 자유롭게 타이핑합니다. 각 chunk에는 시간이 붙습니다.
3. Record필요할 때만 녹음을 켭니다. 녹음은 기본값이 아닙니다.
4. Match메모 시간과 transcript 구간을 맞춰 맥락을 찾습니다.
5. SaveAI 정리본을 Obsidian Markdown으로 저장합니다.

제품 포지션

시장에는 이미 비슷한 앱이 있습니다. 특히 Cleft는 말하거나 타이핑한 내용을 구조화된 노트로 만들고, 로컬 전사와 Obsidian Markdown 저장까지 강조합니다. 따라서 생각결이 “AI voice note app”으로만 가면 약합니다.

생각결의 포지션은 한국어/Obsidian/vault-first 환경에서 “사용자가 직접 남긴 생각의 시간”과 “실제 발화 맥락”을 연결하는 것입니다.

Cleft: voice-first note Granola: meeting notes Superwhisper: dictation AudioPen: polished text 생각결: time-context restoration

Product-Core MVP

첫 버전부터 생각결의 핵심 장면을 넣습니다. 너무 얇은 빠른 노트앱으로 시작하면 제품의 이유가 드러나지 않습니다.

UX 구조

생각결의 기본 플로팅 UI는 복잡한 에디터가 아니라 sticky note처럼 단순한 capture surface입니다. 사용자는 작은 창에서 바로 쓰고, 녹음을 켜고, 세션을 끝낼 수 있습니다. transcript 대조, AI 정리, task 확인처럼 더 넓은 맥락이 필요할 때만 expanded session editor로 확장합니다.

Sticky capture note

전역 단축키로 즉시 뜨는 작은 창입니다. 타이머, 입력창, 녹음 토글, autosave, expand, end session만 보여줍니다. 작지만 내용이 작아지면 안 되므로 스크롤과 점진적 높이 확장을 허용합니다.

Expanded session editor

Sticky note에서 확장되는 큰 작업 표면입니다. Notes / Transcript / Synthesis / Tasks 탭과 transcript evidence, session list, actions를 제공합니다.

로드맵

Phase 1
Capture skeleton

메뉴바 앱, 전역 단축키 sticky capture note, 확장형 세션 에디터, timestamped notes를 만든다.

Phase 2
Obsidian output

세션 종료 시 Markdown을 생성하고, 로컬 DB에 세션 인덱스와 처리 상태를 저장한다.

Phase 3
Recording and transcript

선택 녹음, transcription, transcript chunk 저장, 시간 offset 매칭을 구현한다.

Phase 4
AI synthesis

세션 종료 시 summary, decisions, tasks, questions, evidence snippets를 생성한다.

Phase 5
Widget and portfolio polish

위젯, README, demo video, screenshots, release packaging을 정리한다.

남은 질문

  • Transcription engine은 Apple Speech, local Whisper, cloud transcription 중 무엇으로 시작할까?
  • AI provider는 OpenAI API, 로컬 모델, provider 설정형 중 무엇이 좋을까?
  • Raw audio는 보관, 자동 삭제, 사용자 설정 중 무엇이 기본값이어야 할까?
  • Obsidian 저장 위치는 프로젝트별 폴더, session log 폴더, 별도 생각결 폴더 중 어디가 좋을까?
