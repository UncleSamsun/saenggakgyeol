PRD

Product

메모와 녹음 transcript를 시간순으로 맞춰 생각의 맥락을 복원하는 Mac capture app.

Primary user

Minjoun. 회의, 혼자 생각하는 세션, 프로젝트 정리 중 긴 내용을 흐름대로 쓰는 사용자.

Product promise

세션이 끝났을 때 사용자는 “내가 왜 이 메모를 남겼는지”를 transcript 근거와 함께 다시 볼 수 있다.

Problem

회의나 사고 세션 중에는 메모를 계속 남기지만, 시간이 지나면 문장만 남고 맥락이 사라집니다. 녹음 transcript가 있어도 메모와 따로 있으면 다시 대조하는 비용이 커서 실제로 잘 쓰지 않게 됩니다.

MVP Goal

v1은 menu bar app, global hotkey Sticky Capture Note, expanded session editor, timestamped notes, optional recording, transcript matching, AI synthesis, Obsidian Markdown output까지 이어지는 Product-Core MVP입니다.

Non-goals

  • 전체 노트앱 대체.
  • 항상 켜져 있는 background recording.
  • 실시간 AI 공동작성 또는 입력 중 자동 요약.
  • 팀 공유, 계정 시스템, 웹 동기화.
  • 복잡한 block editor.
  • 위젯 안에서 긴 편집을 처리하는 UX.

Functional Requirements

ID Requirement Acceptance
R1 사용자는 메뉴바에서 새 세션을 시작하고 최근 세션을 열 수 있다. 메뉴바 popover에서 active/recent/pending 상태가 보이고 sticky note 또는 expanded editor로 진입한다.
R2 전역 단축키는 Sticky Capture Note를 500ms 목표로 연다. 이미 active session이 있으면 새 세션이 아니라 기존 sticky note를 복귀한다.
R3 사용자는 sticky note에서 바로 쓰고, 필요하면 expanded editor로 확장할 수 있다. Sticky note는 스크롤/높이 확장을 지원하고, 30분 세션과 100개 이상 note chunk는 expanded editor에서 막히지 않는다.
R4 각 note chunk에는 세션 시작 기준 시간이 붙는다. Markdown과 DB 모두 같은 offset을 보존한다.
R5 녹음은 기본 OFF이고 사용자가 세션 중 켤 수 있다. 녹음 토글은 note timestamp를 깨지 않고 permission/failure 상태를 노출한다.
R6 전사 결과는 transcript chunk로 저장된다. 각 chunk는 start/end offset, text, optional confidence/speaker를 가진다.
R7 note chunk 선택 시 주변 transcript context를 볼 수 있다. 기본 window는 해당 note 기준 전후 60초다.
R8 세션 종료 후 AI synthesis를 실행할 수 있다. summary, decisions, tasks, questions, evidence snippets가 생성되거나 실패 상태가 보인다.
R9 세션 결과는 Obsidian Markdown으로 저장된다. 앱 DB가 없어도 Markdown은 사람이 읽을 수 있다.
R10 로컬 DB는 index/cache/status를 담당한다. DB에는 session, note, transcript, synthesis processing state가 저장된다.

User Flows

1. Start메뉴바 또는 단축키로 세션을 시작한다.
2. WriteSticky note에 바로 쓰고 필요하면 확장한다.
3. Record필요할 때만 녹음을 켠다.
4. Matchnote와 transcript를 시간 기준으로 대조한다.
5. Synthesize세션 종료 후 AI 정리본을 만든다.
6. SaveObsidian Markdown으로 저장하고 열 수 있다.

Primary Flow: Solo Thinking Session

  1. 사용자가 전역 단축키를 누른다.
  2. Sticky Capture Note가 active session 없이 열리면 새 solo session을 만든다.
  3. 사용자는 sticky note에서 바로 작성한다.
  4. 내용이 길어지거나 transcript context가 필요하면 expanded session editor로 확장한다.
  5. 필요하면 recording toggle을 켠다.
  6. 세션 종료 시 앱은 Markdown draft를 만들고 synthesis를 queued 상태로 둔다.
  7. synthesis 완료 후 사용자는 Obsidian에서 결과를 연다.

Secondary Flow: Meeting Session

  1. 사용자가 메뉴바에서 녹음 세션 시작을 선택한다.
  2. permission이 없으면 앱은 recording permission screen으로 안내한다.
  3. 회의 중 사용자는 notes를 짧거나 길게 남긴다.
  4. 세션 종료 후 transcript와 notes가 시간 기준으로 묶인다.
  5. Tasks 탭에서 할 일, 결정, 질문을 확인한다.

Screen Definitions

모든 화면은 Brand / Design System의 Quiet Native Workspace 기준을 따른다. AI 출력은 사용자가 쓴 원문보다 시각적으로 앞서면 안 된다.

S1. Menu Bar Popover

Area Content Behavior
Header 생각결 logo, active session timer, recording state. Active session이 있으면 가장 위에 고정한다.
Quick actions Start quick note, start recording session, resume session. Primary action은 active session 유무에 따라 resume 또는 start.
Recent sessions 최근 5개 session title, status, last activity. 클릭 시 sticky note 또는 expanded session editor로 연다.
Footer Open Obsidian, Settings, processing failures. 실패가 있으면 warning state를 노출한다.

S2. Sticky Capture Note

기본 플로팅 UI는 sticky note처럼 단순해야 한다. 사용자는 작은 창에서 생각을 바로 적고, 녹음 상태를 확인하고, 필요하면 큰 세션 에디터로 확장한다.

  • Window는 draggable이고, always-on-top 설정을 지원할 수 있다.
  • Chrome은 최소화한다. title, timer, autosave state, recording toggle, expand, end session만 유지한다.
  • 작은 UI여도 텍스트 크기는 작아지면 안 된다. 입력 영역은 스크롤과 점진적 높이 확장을 지원한다.
  • Sticky note에서 작성된 모든 note chunk는 expanded editor와 같은 session에 저장된다.

Context

03:05-03:55

선택된 note 주변 transcript가 있으면 이 영역에 표시한다.

  • Sticky note에서 expand하면 같은 active session을 유지한 채 열린다.
  • Compact width에서는 session list와 context panel을 접고 Notes editor를 먼저 지킨다.
  • Expanded width 목표는 760px 이상이다.
  • Autosave 상태는 조용하게 표시한다. 사용자를 방해하지 않는다.
  • End Session은 destructive하지 않다. 종료 후에도 session은 다시 열 수 있다.

S4. Session Editor Tabs

Tab Purpose Empty state Primary action

S5. Settings

  • Vault path: 기본값은 `/Users/minjoun/Documents/DevKnowledge`로 시작한다.
  • Global hotkey: 기본값은 구현 시 충돌 검사를 거쳐 정한다.
  • Recording permission: microphone/system audio permission 상태를 표시한다.
  • AI provider: v1 UI는 provider 준비 상태만 표시할 수 있다. 실제 provider 결정은 ADR에서 확정한다.
  • Audio retention: v1 UI는 정책 준비 상태만 표시한다. 기본값은 ADR에서 확정한다.

S6. Widget

Widget은 core flow 이후 단계다. v1 문서에는 상태 표시와 빠른 진입 요구만 남긴다. 긴 작성, 녹음 permission, 편집은 Sticky Capture Note 또는 expanded session editor에서 처리한다.

States and Errors

State User sees Required handling
Recording permission denied 녹음 권한이 없어 녹음을 시작할 수 없다는 안내. Notes 작성은 계속 가능해야 한다.
Transcription failed Transcript 탭에 실패 상태와 retry/import option. Raw notes와 audio reference는 보존한다.
AI synthesis failed Synthesis 탭에 retry와 실패 이유. Markdown draft는 notes 중심으로라도 생성한다.
Obsidian write failed 저장 경로 문제와 retry action. Local session state는 completed가 아니라 failed로 남긴다.
Active session interrupted 다시 열었을 때 autosaved notes와 timer status. 마지막 saved note chunk 이후부터 이어 쓸 수 있어야 한다.

Acceptance Criteria

  • 전역 단축키 또는 메뉴바에서 active session의 Sticky Capture Note를 열 수 있다.
  • 30분 이상 세션에서 긴 notes를 작성하고 timestamp가 유지된다.
  • Sticky Capture Note에서 작성한 note chunk가 expanded session editor에 같은 session으로 이어진다.
  • 녹음 OFF 상태에서도 핵심 notes flow가 완성된다.
  • 녹음 ON 상태에서는 transcript chunk가 note chunk와 시간 기준으로 매칭된다.
  • note chunk 선택 시 전후 60초 transcript context를 볼 수 있다.
  • 세션 종료 후 Markdown draft가 생성된다.
  • AI synthesis 실패, transcription 실패, Obsidian write 실패가 각각 별도 상태로 보인다.
  • Obsidian Markdown은 앱 DB 없이도 사람이 읽을 수 있다.
  • 화면은 Brand / Design System의 token, 색상, typography 원칙을 따른다.
